# C PLUS PLUS

## 1. 项目概述

本仓库为 C++ 课程与实践代码集合。根目录下大量 `.txt` 文件用于记录各阶段语法与示例说明；`课程代码` 目录按章节组织控制台示例。其中较完整的小型应用为控制台“自动售货/超市”模拟：`main.cpp` 从 `food.txt` 初始化货架，用户在循环中输入商品名称加入购物车，输入 `exit` 结束。

## 2. 技术栈

### 前端

无（控制台交互，无 Web 前端）。

### 后端

- C++（控制台程序、文件流读写、`iostream` 交互）

### 数据库

无独立数据库；商品数据通过文本文件（如 `food.txt`）加载。

### 开发工具与运维

- Visual Studio 或 MSVC 工程痕迹（部分子目录含 `.sln`/`.vcxproj` 或构建产物）
- 文本编辑器保存的课堂笔记（`.txt`）

## 3. 项目架构

整体为**多示例并列**结构，彼此无统一“框架”耦合：

- **笔记层**：根目录 `.txt` 记录知识点与片段。
- **课程示例层**：`课程代码` 下按 `ch2`、`ch3`… 分章目录，每章多为单一 `main.cpp` 入口。
- **综合小项目层**：`自动售货系统搭建（上）：控制台/SelfShop` 将 `Shelf`、`Cart`、`Food` 等类型组合，`main` 负责命令行会话与读文件初始化。

数据流：`food.txt` → `initShelf` 读入 → `Shelf` 维护商品 → 用户输入商品名 → `addCart` 扣减库存并更新 `Cart` → `display` 打印货架与购物车。

下列树省略 `.git`、`__pycache__`、`node_modules`、`.venv`、`venv`、`dist` 等常见依赖与构建目录，仅列出与本仓库学习目的相关的部分：

## 5. 核心文件说明

### 项目入口文件和配置文件

- `课程代码/自动售货系统搭建（上）：控制台/SelfShop/src/main.cpp`：程序入口；定义 `initShelf`、`display`、`addCart`，命令行循环直至用户输入 `exit`。
- `课程代码/ch2/ch2-HelloWorld/src/main.cpp`：章节示例入口，从标准输入读入名字并输出问候。
- `food.txt`（位于 `SelfShop` 目录下，与 `main.cpp` 中 `open("food.txt")` 对应）：商品列表数据源。

### 核心业务逻辑实现

- `main.cpp` 内 `initShelf`：使用 `std::ifstream` 逐行读取名称、价格、数量、保质期，向 `Shelf` 添加 `Food` 实例。
- `main.cpp` 内 `addCart`：按名称查找 `Item`，校验库存后扣减并入队购物车。
- `main.cpp` 内 `display`：遍历货架与购物车并格式化输出价格与数量。

### 数据模型和 API 接口

- 无 HTTP/RPC；数据模型为 C++ 类（如 `Shelf`、`Cart`、`Food` 等，声明于同目录头文件/源文件）。对外“接口”即控制台输入与文件格式约定。

### 关键组件和服务模块

- **货架模块**：`Shelf` 管理多类 `Item`。
- **购物车模块**：`Cart` 汇总选购数量。
- **持久化**：通过文本文件初始化货架；运行时变更主要在内存中。

根目录中少量 `.py` 文件与 PDF 等为辅助材料，与核心 C++ 控制台主线相对独立。
